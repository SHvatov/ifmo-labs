Cсылка на задание

Значения параметров, при которых возможнlо переполнение стека вызовов зависит от того, как именно будет определена функция pwr, а именно ее граничное условие.

Вариант №1 

const pwr = (a, b) => (b === 1) ? a : mpy(a, pwr(a, b - 1));
В данном случае, переполнение стека происходит при любых значениях параметра b < 1, так как граничное условие в таком случае не срабатывает, и происходит вызов pwr(N, 1) -> pwr(N, 0) -> pwr(N, -1) -> .... На локальной машине переполнение стека происходит при значении аргумента b = -8373. При этом поток выполнения никогда не доходит до других функций.

Вариант №2 
const pwr = (a, b) => (b === 1) ? a : (b === 0 ? 1 : mpy(a, pwr(a, b - 1)));
В данном случае предусмотрен вариант, когда значение b == 0, но переполнение стека все равно происходит при любых значениях параметра b < 0, так как граничное условие в таком случае не срабатывает, и происходит вызов pwr(N, -1) -> pwr(N, -2) -> pwr(N, -3) -> ....

В обоих случаях также возникает переполнение стека если вызвать функцию с отрицательным или нулевым значением аргумента a. Рассмотрим в качестве примера вызов pwr(-2, 2). Стек вызова будет следующим: pwr -2 2 -> pwr -2 1 -> mpy -2 -2 -> mpy -2 -3 -> mpy -2 -4 -> ...
